## What this project is
This repository contains the UX redesign prototype of the Study Time application.

- Study Time V2 (`https://www.studytime.biz/`) is the live, ready-to-use production system.
- Study Time V3 (`https://study-time-mockup-prototype.vercel.app/`) is a prototype-focused redesign used for design validation and stakeholder feedback.
- In this prototype, only AI features are connected to real API endpoints.
- Most other app data is prototype/mock data and is not connected to a production database.

## Study Time Application Redesign - UX Research & Design Project

### Project overview
This redesign improves the existing Study Time experience across desktop, tablet, and mobile.
It keeps core capabilities such as study session logging, timers, journaling, and search, while adding:
- Guided reflection questions
- AI-supported analytics insights
- Dark and light modes
- Topic and hashtag management
- Responsive design
- A dashboard for quick access to recent sessions and key statistics
- Study activity visualized in daily, weekly, and monthly calendar views

### UX design workflow
1. Project brief and user research  
Stakeholder questionnaires defined scope, interface style, and priorities. Competitor analysis (including Athenify and MyStudyLife) and user research highlighted the need for low-effort tracking, better data retrieval, and AI assistance.

2. Concept design: sketches and wireframes  
Storyboards, sketches, and wireframes were produced for Dashboard, Log Session, Session History, and Analytics. Early feedback confirmed direction and requested clearer labels and controls, especially on Log Session.

3. Visual design and prototyping  
The redesign targeted WCAG 2.2 Level AA accessibility, with readable typography and a neutral visual system. A high-fidelity interactive prototype was implemented directly in code for realistic evaluation.

4. User testing and feedback  
Stakeholders rated Dashboard, Session History, and Analytics highly. Log Session scored lower due to flow and layout issues, especially timer placement.

5. Iteration and final delivery  
Four Log Session alternatives were tested. Concept D was selected because it reduced friction by moving the Topic controls directly into the main logging flow. Final refinements included better mobile icon spacing and a missing analytics chart Y-axis, followed by stakeholder sign-off.
